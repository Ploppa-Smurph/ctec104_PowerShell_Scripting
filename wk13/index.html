Major Unix Shells, Regular Expressions, Scripting Commands
history of Unix Shells: 1977= Bourne Shell; 1978= C Shell; 1983= Korn Shell; and 1989 = BASH

Key Characteristics of Unix / Linux Shell

  • Designed to work with the file system
  • NO scripting support for: processes, system logs, or system configuration -- all of those are defined in text based files
  • Unix Shells do not have any object data types -- All commands are text-based
  • NO built in system security for shell scripts or 'profiles'

Profile

the 'profile' was created for Unix shells -- it is a text file stored in the user's home directory
Profile file stores initial configuration for the user's shell environment & user's personal aliases
commands in the Profile run every time a user opens a new Shell

Major Unix Shells

Bourne Shell named after creator Stephen Bourne, was the 1st modern Shell (1977)
Bourne Shells were the first to offer:
  • Shell scripts could use pipelined execution
  • Control Flow with 'if' conditions and looping
  • Variables for user data
  • Environment variables that define operating characteristics for the shell
C Shell was next in 1978
  • syntax closely resembles C programming language
  • added: Command History
  • added: Aliases
  • added: Command Completion
  • added: Job Control
Korn Shell developed in 1983
Korn was backwards compatible with Bourne Shell and incorporated the new shell features of C shell
  • job control
  • aliases
  • command history
  • added: Associative Arrays for data storage
BASH (Bourne Again SHell) was introduced in 1989
-- BASH contains all features developed after the 1st Bourne Release and is backwards compatible with Bourne Shell
BASH is the most popular of available shells

Regular Expressions

an essential part of scripting in linux is the text-based restriction for commands
Developers have created advanced ways to manipulate text data
    Some Commands are
  • grep
  • sed
  • awk
The basis of all text manipulation commands is regular expressions
Regular Expression is a sequence of characters used to define a search pattern
these search patterns are used to identify information like:
  • File and Directory names
  • Lines of data inside a text file
  • Output of a command that is pipelined to another command
Regular expressions work like wildcards but with more options

Common Regular Expression Metacharacters

[] brackets are used to match a single character for a set:
ex: file[1-9] = file1, file2, file3, file4, file5, file6, file7, file8, file9
ex02: file[abc] = filea, fileb, filec

. period (dot) is used to match a single character
ex: wo.d == wood, word, wo2d, woTd, etc...

* asterisk is used to match the previous value zero or more times
ex: ab*c translates to ac, abc, abbc, abbbc, abbbbc, etc...
there are many more Regular Expressions and they can look indecipherable at times

Scripting Commands

grep = Globally find a Regular Expression and Print (GREP)
this command searches a text file / or pipelined input and prints all the lines containing the text string
grep [options] pattern [file]
grep output can be: Displayed to Screen, Saved in an Output File, or Piped to another command

ex: grep tcp services -- this will search the 'services' file and look for lines containing 'tcp'
ex02: ls | grep 20211215 == receives a directory listing as input and displays all of the files with the given date as part of their name

sed == 'stream editor' this command will read lines of text from a text file / input stream and perform operations on the text
sed has around 25 commands to perform various text manipulations
output of sed can be: displayed to screen, saved as output, piped to another command
sed you create script files to store a set of command operation to perform on the input

ex: sed 's/Nick/John/g' report.txt > report_new.txt -- performs a global search and replaces the name in the report file
ex02: sed -n 12,18p file.txt -- displays lines 12 -18 of file.txt

awk - is an abbreviation of the creators last names. 'awk' has more advanced manipulation features than sed
awk supports variables, arrays, functions, and programming control structures
awk is considered it's own scripting language and is much more complex. It is an example of how programmers managed data before modern shells like PowerShell.
